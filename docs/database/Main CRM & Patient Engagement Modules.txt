# CATEGORY 20 — CRM & PATIENT ENGAGEMENT MANAGEMENT SYSTEM

## Module Overview

The CRM (Customer Relationship Management) & Patient Engagement System manages the complete patient relationship lifecycle from lead generation to long-term patient retention.

This module helps hospitals:

* Capture leads
* Convert leads into patients
* Run marketing campaigns
* Manage enquiries
* Handle call center operations
* Send notifications
* Collect feedback
* Resolve complaints
* Manage loyalty programs
* Track referrals
* Run follow-up campaigns
* Measure patient engagement
* Improve patient retention

---

# 1. LEAD MANAGEMENT

## TABLE: lead_sources

### Purpose

Stores sources from which leads originate.

### Examples

* Website
* Google Ads
* Facebook
* Referral
* Walk-In

| Column         | Data Type    | Description        |
| -------------- | ------------ | ------------------ |
| lead_source_id | UUID         | Primary Key        |
| source_code    | VARCHAR(50)  | Unique source code |
| source_name    | VARCHAR(255) | Source name        |
| created_at     | TIMESTAMP    | Record creation    |

---

## TABLE: lead_statuses

### Purpose

Stores lead lifecycle statuses.

### Examples

* New
* Contacted
* Qualified
* Converted
* Lost

| Column         | Data Type    | Description   |
| -------------- | ------------ | ------------- |
| lead_status_id | UUID         | Primary Key   |
| status_code    | VARCHAR(50)  | Status code   |
| status_name    | VARCHAR(255) | Status name   |
| created_at     | TIMESTAMP    | Creation date |

---

## TABLE: patient_leads

### Purpose

Stores prospective patient information.

### Relationships

* Belongs to lead source
* Belongs to lead status
* Can be assigned to CRM staff

| Column         | Data Type    | Description   |
| -------------- | ------------ | ------------- |
| lead_id        | UUID         | Primary Key   |
| lead_source_id | UUID         | Lead source   |
| lead_status_id | UUID         | Lead status   |
| first_name     | VARCHAR(255) | First name    |
| last_name      | VARCHAR(255) | Last name     |
| email          | VARCHAR(255) | Email         |
| mobile_number  | VARCHAR(50)  | Mobile        |
| created_at     | TIMESTAMP    | Creation date |

---

## TABLE: lead_assignments

### Purpose

Assigns leads to staff members.

| Column               | Data Type | Description     |
| -------------------- | --------- | --------------- |
| assignment_id        | UUID      | Primary Key     |
| lead_id              | UUID      | Lead reference  |
| assigned_employee_id | UUID      | Employee        |
| assigned_date        | TIMESTAMP | Assignment date |

---

## TABLE: lead_activities

### Purpose

Tracks lead interactions.

### Examples

* Phone calls
* Emails
* Follow-ups
* Meetings

| Column         | Data Type    | Description    |
| -------------- | ------------ | -------------- |
| activity_id    | UUID         | Primary Key    |
