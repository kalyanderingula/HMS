# Enterprise Hospital Management System (HMS)

# Appointment Management Module Documentation

---

# 1. Module Overview

The Appointment Management module is the operational scheduling engine of the hospital management system.

This module manages:

* Appointment booking
* Doctor scheduling
* Slot management
* Queue management
* Check-ins
* Follow-ups
* Telemedicine appointments
* Appointment reminders
* Rescheduling
* Waitlists
* Appointment billing integration
* Appointment audit tracking

This module integrates with:

* Patient Management
* Doctor Management
* Department Management
* Billing & Payments
* EMR/EHR
* Telemedicine
* Notification System
* Queue Management

---

# 2. Enterprise Architecture Principles

This schema follows enterprise-grade standards:

* UUID-based primary keys

* Multi-doctor scheduling support

# 4. Core Appointment Tables

---

# TABLE: appointments

## Purpose

This is the root table of the Appointment Management module.

## Why This Table Exists

Stores all patient appointment records.

## Relationships

* One patient can have many appointments.
* One doctor can handle many appointments.
* One department can manage many appointments.
* One appointment can have many reminders, notes, and documents.

---

| Column Name                | Data Type    | Description                  |
| -------------------------- | ------------ | ---------------------------- |
| appointment_id             | UUID         | Primary key                  |
| tenant_id                  | UUID         | Multi-tenant identifier      |
| appointment_number         | VARCHAR(100) | Unique appointment number    |
| patient_id                 | UUID         | FK to patients               |
| doctor_id                  | UUID         | FK to doctors                |
| department_id              | UUID         | FK to departments            |
| appointment_type_id        | UUID         | FK to appointment_types      |
| appointment_status_id      | UUID         | FK to appointment_statuses   |
| priority_id                | UUID         | FK to appointment_priorities |
| appointment_date           | DATE         | Appointment date             |
| start_time                 | TIME         | Appointment start            |
| end_time                   | TIME         | Appointment end              |
| estimated_duration_minutes | INT          | Duration estimate            |
| chief_complaint            | TEXT         | Patient complaint            |
| notes                      | TEXT         | Additional notes             |
| is_first_visit             | BOOLEAN      | First visit indicator        |
| requires_followup          | BOOLEAN      | Follow-up requirement        |
| booking_source             | VARCHAR(100) | Online/Walk-in/Call          |
| booked_by                  | UUID         | Booking user                 |
| booked_at                  | TIMESTAMP    | Booking timestamp            |
| confirmed_at               | TIMESTAMP    | Confirmation time            |
| checked_in_at              | TIMESTAMP    | Check-in time                |
| completed_at               | TIMESTAMP    | Completion time              |
| cancelled_at               | TIMESTAMP    | Cancellation time            |
| cancellation_reason_id     | UUID         | FK to cancellation_reasons   |
| created_at                 | TIMESTAMP    | Creation timestamp           |
| updated_at                 | TIMESTAMP    | Update timestamp             |
| created_by                 | UUID         | Created user                 |
| updated_by                 | UUID         | Updated user                 |
| deleted_at                 | TIMESTAMP    | Soft delete timestamp        |
