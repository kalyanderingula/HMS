# Enterprise Hospital Management System (HMS)

# Ambulance & Transport Management System Documentation

---

# 1. Module Overview

The Ambulance & Transport Management System is the hospital’s emergency transportation, fleet operations, dispatch coordination, GPS tracking, patient transport, and emergency response backbone.

This module manages:

* Ambulance fleet management
* Emergency dispatch coordination
* GPS and telematics tracking
* Driver and paramedic scheduling
* Patient transport workflows
* Interfacility transfers
* Trip billing and insurance claims
* Ambulance inventory tracking
* Fleet maintenance management
* Safety and incident tracking
* Regulatory compliance
* Emergency response analytics
* Transport KPI monitoring
* Audit and compliance tracking

This module integrates with:

* Emergency Department
* ICU & Critical Care
* Billing & Financial Systems
* Insurance Claims
* GPS & Maps APIs
* IoT Telematics Devices
* Mobile Applications
* Government Emergency Networks
* Fleet Tracking Platforms

---

# 2. Enterprise Architecture Principles

This schema follows enterprise fleet and emergency transport standards:

* UUID-based primary keys
* Real-time GPS readiness
* Fleet scalability
* Emergency response optimization
* Compliance-ready architecture
* High-volume dispatch support
* Predictive maintenance readiness
* Multi-city ambulance operations
* IoT telematics integration
* Audit-ready transport tracking

# TABLE: ambulance_types

## Purpose

Stores ambulance classification types.

## Example Types

* Basic Life Support (BLS)
* Advanced Life Support (ALS)
* Neonatal Ambulance
* ICU Ambulance

| Column Name       | Data Type    | Description        |
| ----------------- | ------------ | ------------------ |
| ambulance_type_id | UUID         | Primary key        |
| type_code         | VARCHAR(100) | Type code          |
| type_name         | VARCHAR(255) | Ambulance type     |
| emergency_level   | VARCHAR(100) | Emergency level    |
| description       | TEXT         | Description        |
| created_at        | TIMESTAMP    | Creation timestamp |

# TABLE: ambulances

## Purpose

Stores ambulance fleet master data.

## Why This Table Exists

Acts as the central fleet registry.

## Relationships

* One ambulance can have multiple devices.
* One ambulance can have maintenance logs.
* One ambulance can have trip records.
* One ambulance can have GPS tracking.

| Column Name         | Data Type    | Description                 |
| ------------------- | ------------ | --------------------------- |
| ambulance_id        | UUID         | Primary key                 |
| ambulance_type_id   | UUID         | FK to ambulance_types       |
| vehicle_number      | VARCHAR(100) | Unique vehicle number       |
| registration_number | VARCHAR(100) | Registration number         |
| chassis_number      | VARCHAR(255) | Chassis number              |
| manufacturer_name   | VARCHAR(255) | Manufacturer                |
| model_name          | VARCHAR(255) | Vehicle model               |
| manufacturing_year  | INT          | Manufacturing year          |
| current_status      | VARCHAR(100) | Current availability status |
| gps_enabled         | BOOLEAN      | GPS enabled flag            |
| created_at          | TIMESTAMP    | Creation timestamp          |
