# Enterprise Hospital Management System (HMS)

# Laboratory Information System (LIS) Documentation

---

# 1. Module Overview

The Laboratory Information System (LIS) is the diagnostic backbone of the hospital management system.

This module manages:

* Laboratory test master data
* Lab orders
* Sample collection
* Specimen tracking
* Result processing
* Pathology workflows
* Microbiology workflows
* Instrument integration
* Quality control
* Laboratory billing
* Inventory and reagents
* Audit tracking

This module integrates with:

* Patient Management
* Doctor Management
* EMR/EHR
* Billing
* Insurance
* Radiology
* Pharmacy
* External analyzers
* HL7/FHIR systems

---

# 2. Enterprise Architecture Principles

This schema follows enterprise laboratory standards:

* UUID-based primary keys
* Normalized diagnostic architecture
* Multi-department laboratory support
* Barcode sample tracking
* Instrument connectivity
* Result auditability
* Quality control workflows
* Enterprise scalability
* Multi-branch pathology support
* Compliance-ready architecture

# 3. Laboratory Master Tables

---

# TABLE: lab_departments

## Purpose

Stores laboratory department information.

## Why This Table Exists

Different laboratory domains require separate workflows.

## Example Departments

* Hematology
* Biochemistry
* Microbiology
* Pathology
* Cytology

| Column Name       | Data Type    | Description            |
| ----------------- | ------------ | ---------------------- |
| lab_department_id | UUID         | Primary key            |
| department_code   | VARCHAR(100) | Unique department code |
| department_name   | VARCHAR(255) | Department name        |
| description       | TEXT         | Department details     |
| department_head   | UUID         | Department head        |
| created_at        | TIMESTAMP    | Creation timestamp     |

---

# TABLE: lab_tests

## Purpose

Stores all laboratory tests.

## Why This Table Exists

This is the core master table for diagnostic tests.

## Example Tests

* CBC
* Liver Function Test
* HbA1c
* Thyroid Profile

| Column Name           | Data Type     | Description                  |
| --------------------- | ------------- | ---------------------------- |
| test_id               | UUID          | Primary key                  |
| test_code             | VARCHAR(100)  | Unique test code             |
| test_name             | VARCHAR(255)  | Test name                    |
| category_id           | UUID          | FK to lab_test_categories    |
| subcategory_id        | UUID          | FK to lab_test_subcategories |
| lab_department_id     | UUID          | FK to lab_departments        |
| test_method           | VARCHAR(255)  | Diagnostic method            |
| turnaround_time_hours | INT           | Result delivery time         |
| sample_volume         | VARCHAR(100)  | Required sample quantity     |
| fasting_required      | BOOLEAN       | Fasting requirement          |
| is_active             | BOOLEAN       | Active status                |
| price                 | NUMERIC(14,2) | Test price                   |
| created_at            | TIMESTAMP     | Creation timestamp           |

# 5. Laboratory Orders

# TABLE: lab_orders

## Purpose

This is the root table of LIS operations.

## Why This Table Exists

Stores all laboratory orders.

## Relationships

* One order can contain multiple tests.
* One order can generate multiple samples.
* One order can generate multiple results.

| Column Name         | Data Type    | Description                |
| ------------------- | ------------ | -------------------------- |
| lab_order_id        | UUID         | Primary key                |
| order_number        | VARCHAR(100) | Unique order number        |
| patient_id          | UUID         | FK to patients             |
| encounter_id        | UUID         | FK to patient_encounters   |
| doctor_id           | UUID         | FK to doctors              |
| lab_order_status_id | UUID         | FK to lab_order_statuses   |
| priority_id         | UUID         | FK to lab_order_priorities |
| ordered_at          | TIMESTAMP    | Order timestamp            |
| clinical_notes      | TEXT         | Clinical details           |
| created_by          | UUID         | Ordering user              |
| created_at          | TIMESTAMP    | Creation timestamp         |

---

# TABLE: lab_order_items

## Purpose

Stores ordered tests and panels.

## Why This Table Exists

One order can contain multiple laboratory investigations.

| Column Name   | Data Type    | Description       |
| ------------- | ------------ | ----------------- |
| order_item_id | UUID         | Primary key       |
| lab_order_id  | UUID         | FK to lab_orders  |
| test_id       | UUID         | FK to lab_tests   |
| panel_id      | UUID         | FK to lab_panels  |
| order_status  | VARCHAR(100) | Item status       |
| ordered_at    | TIMESTAMP    | Ordered timestamp |
