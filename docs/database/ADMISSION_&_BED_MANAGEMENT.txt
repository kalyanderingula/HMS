# Enterprise Hospital Management System (HMS)

# Admission & Bed Management Module Documentation

---

# 1. Module Overview

The Admission & Bed Management module is the inpatient operational core of the hospital management system.

This module manages:

* Patient admissions
* Ward management
* Room management
* Bed allocation
* Bed transfers
* Room maintenance
* Bed maintenance
* Discharge planning
* Inpatient billing
* Admission notes
* Admission audit tracking

This module integrates with:

* Patient Management
* Doctor Management
* Department Management
* EMR/EHR
* Billing
* Nursing
* ICU Management
* Insurance
* Pharmacy
* Laboratory

---

# 2. Enterprise Architecture Principles

This schema follows enterprise-grade hospital standards:

* UUID-based primary keys
* Fully normalized relational design
* Real-time bed tracking
* Ward hierarchy support
* Transfer workflow support
* Audit logging
* Change history tracking
* Multi-tenant architecture
* High-volume inpatient scalability
* Compliance-ready architecture

# TABLE: admission_statuses

## Purpose

Stores patient admission lifecycle statuses.

## Why This Table Exists

Admissions move through multiple operational states.

## Example Values

* Admitted
* Under Observation
* Discharged
* Transferred
* Cancelled

---

| Column Name         | Data Type    | Description            |

# 4. Core Infrastructure Tables

---

# TABLE: wards

## Purpose

Stores hospital ward information.

## Why This Table Exists

Wards organize rooms and beds into operational inpatient areas.

## Example Wards

* ICU Ward
* Cardiology Ward
* Pediatric Ward
* Emergency Ward

---

| Column Name   | Data Type    | Description        |
| ------------- | ------------ | ------------------ |
| ward_id       | UUID         | Primary key        |
| department_id | UUID         | FK to departments  |
| ward_code     | VARCHAR(100) | Unique ward code   |
| ward_name     | VARCHAR(255) | Ward name          |
| floor_number  | VARCHAR(50)  | Floor location     |
| building_name | VARCHAR(255) | Building name      |
| total_rooms   | INT          | Number of rooms    |
| total_beds    | INT          | Number of beds     |
| ward_status   | VARCHAR(100) | Operational status |
| created_at    | TIMESTAMP    | Creation timestamp |
| updated_at    | TIMESTAMP    | Update timestamp   |

---

# TABLE: rooms

## Purpose

Stores room information inside wards.

## Why This Table Exists

Patients are admitted into rooms before specific bed assignment.

## Relationship

* One ward can have multiple rooms.
* One room can contain multiple beds.

---

| Column Name       | Data Type     | Description        |
| ----------------- | ------------- | ------------------ |
| room_id           | UUID          | Primary key        |
| ward_id           | UUID          | FK to wards        |
| room_number       | VARCHAR(100)  | Unique room number |
| room_type_id      | UUID          | FK to room_types   |
| room_status       | VARCHAR(100)  | Room status        |
| floor_number      | VARCHAR(50)   | Floor number       |
| capacity          | INT           | Bed capacity       |
| current_occupancy | INT           | Current occupancy  |
| daily_charge      | NUMERIC(12,2) | Daily room charge  |
| created_at        | TIMESTAMP     | Creation timestamp |
| updated_at        | TIMESTAMP     | Update timestamp   |

---

# TABLE: admissions

## Purpose

This is the root table of the Admission Management module.

## Why This Table Exists

Stores all inpatient admission records.

## Relationships

* One patient can have many admissions.
* One admission belongs to one ward, room, and bed.
* One admission can have multiple notes and documents.

---

| Column Name               | Data Type    | Description                |
| ------------------------- | ------------ | -------------------------- |
| admission_id              | UUID         | Primary key                |
| tenant_id                 | UUID         | Multi-tenant identifier    |
| admission_number          | VARCHAR(100) | Unique admission number    |
| patient_id                | UUID         | FK to patients             |
| encounter_id              | UUID         | FK to patient_encounters   |
| admission_type_id         | UUID         | FK to admission_types      |
| admission_status_id       | UUID         | FK to admission_statuses   |
| admitting_doctor_id       | UUID         | FK to doctors              |
| department_id             | UUID         | FK to departments          |
| ward_id                   | UUID         | FK to wards                |
| room_id                   | UUID         | FK to rooms                |
| bed_id                    | UUID         | FK to beds                 |
| admission_reason          | TEXT         | Admission reason           |
| admission_date            | TIMESTAMP    | Admission timestamp        |
| expected_discharge_date   | DATE         | Expected discharge         |
| actual_discharge_date     | TIMESTAMP    | Actual discharge           |
| discharge_summary         | TEXT         | Clinical discharge summary |
| discharge_condition       | TEXT         | Patient condition          |
| referred_by               | VARCHAR(255) | Referral source            |
| emergency_contact_name    | VARCHAR(255) | Emergency contact          |
| emergency_contact_phone   | VARCHAR(20)  | Contact phone              |
| insurance_approval_status | VARCHAR(100) | Insurance state            |
| notes                     | TEXT         | Additional notes           |
| created_at                | TIMESTAMP    | Creation timestamp         |
| updated_at                | TIMESTAMP    | Update timestamp           |
| created_by                | UUID         | Created user               |
| updated_by                | UUID         | Updated user               |
| deleted_at                | TIMESTAMP    | Soft delete timestamp      |

# 5. High-Level Relationship Structure

```text
wards
│
└── rooms
      │
      └── beds
             │
             ├── bed_allocations
             ├── bed_transfers
             └── bed_maintenance

admissions
│
├── patients
├── doctors
├── departments
├── wards
├── rooms
├── beds
├── admission_documents
├── admission_notes
├── discharge_plans
├── admission_billing
├── admission_audit_logs
└── admission_change_history
```
