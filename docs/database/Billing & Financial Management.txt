# Enterprise Hospital Management System (HMS)

# Billing & Financial Management Module Documentation

---

# 1. Module Overview

The Billing & Financial Management module is the financial core of the hospital management system.

This module manages:

* Patient billing
* Invoice generation
* Invoice itemization
* Payments
* Refunds
* Insurance claims
* Credit notes
* Debit notes
* Revenue center accounting
* Financial transactions
* Billing audit tracking
* Revenue cycle management

This module integrates with:

* Patient Management
* Appointment Management
* Admission Management
* EMR/EHR
* Pharmacy
* Laboratory
* Radiology
* Insurance
* Accounting Systems
* Taxation Systems

---

This schema follows enterprise-grade financial architecture standards:

# 4. Core Billing Tables

---

# TABLE: billing_accounts

## Purpose

Stores patient financial account information.

## Why This Table Exists

Each patient requires a centralized financial account.

## Relationships

* One patient can have one or multiple billing accounts.
* One billing account can have multiple invoices.

---

| Column Name             | Data Type     | Description           |
| ----------------------- | ------------- | --------------------- |
| billing_account_id      | UUID          | Primary key           |
| patient_id              | UUID          | FK to patients        |
| account_number          | VARCHAR(100)  | Unique account number |
| account_status          | VARCHAR(100)  | Account state         |
| total_due               | NUMERIC(14,2) | Outstanding balance   |
| total_paid              | NUMERIC(14,2) | Total paid amount     |
| insurance_provider      | VARCHAR(255)  | Insurance company     |
| insurance_policy_number | VARCHAR(255)  | Policy number         |
| credit_limit            | NUMERIC(14,2) | Approved credit limit |
| created_at              | TIMESTAMP     | Creation timestamp    |
| updated_at              | TIMESTAMP     | Update timestamp      |

---

# TABLE: invoices

## Purpose

This is the root table of the Billing module.

## Why This Table Exists

Stores all billing invoices generated by hospital services.

## Relationships

* One invoice can have multiple invoice items.
* One invoice can have multiple payments.
* One invoice can have insurance claims.

---

| Column Name        | Data Type     | Description              |
| ------------------ | ------------- | ------------------------ |
| invoice_id         | UUID          | Primary key              |
| tenant_id          | UUID          | Multi-tenant identifier  |
| invoice_number     | VARCHAR(100)  | Unique invoice number    |
| billing_account_id | UUID          | FK to billing_accounts   |
| patient_id         | UUID          | FK to patients           |
| admission_id       | UUID          | FK to admissions         |
| appointment_id     | UUID          | FK to appointments       |
| encounter_id       | UUID          | FK to patient_encounters |
| invoice_type_id    | UUID          | FK to invoice_types      |
| billing_status_id  | UUID          | FK to billing_statuses   |
| invoice_date       | TIMESTAMP     | Invoice timestamp        |
| due_date           | DATE          | Payment due date         |
| subtotal_amount    | NUMERIC(14,2) | Amount before tax        |
| tax_amount         | NUMERIC(14,2) | Total tax                |
| discount_amount    | NUMERIC(14,2) | Total discount           |
| total_amount       | NUMERIC(14,2) | Final invoice amount     |
| paid_amount        | NUMERIC(14,2) | Paid amount              |
| balance_amount     | NUMERIC(14,2) | Remaining balance        |
| notes              | TEXT          | Additional notes         |
| created_by         | UUID          | User creating invoice    |
| created_at         | TIMESTAMP     | Creation timestamp       |
| updated_at         | TIMESTAMP     | Update timestamp         |

---
