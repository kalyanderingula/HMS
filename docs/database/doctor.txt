# Enterprise Hospital Management System (HMS)

# Doctor Management Module Documentation

---

# 1. Module Overview

The Doctor Management module manages all doctor-related information inside the hospital ecosystem.

This module is responsible for:

* Doctor registration
* Professional profiles
* Specializations
* Department assignments
* Qualifications
* Licensing
* Availability scheduling
* Consultation fee configuration
* Performance tracking
* Audit logging
* Compliance tracking

This module integrates with:

* Appointments
* EMR/EHR
* Admissions
* Billing
* Surgery
* Laboratory
* Radiology
* HR & Payroll

---

# 2. Enterprise Design Principles

This schema follows enterprise-grade architecture principles:

* UUID primary keys
* Fully normalized schema
* Multi-specialization support
* Multi-department support
* Audit logging
* Change tracking
* Soft delete support
* Multi-tenant readiness
* Scalable indexing
* Legal compliance support

# TABLE: doctor_statuses

## Purpose

Stores doctor account statuses.

## Why This Table Exists

Instead of storing status text repeatedly, normalization improves consistency and maintainability.

## Example Values

* Inactive

# 4. Core Doctor Tables

---

# TABLE: doctors

## Purpose

This is the root table of the Doctor Management module.

## Why This Table Exists

Stores core doctor identity and operational information.

## Relationships

* One doctor can have multiple qualifications.
* One doctor can belong to multiple departments.
* One doctor can have multiple specializations.
* One doctor can have multiple schedules.

---

| Column Name                   | Data Type    | Description             |
| ----------------------------- | ------------ | ----------------------- |
| doctor_id                     | UUID         | Primary key             |
| tenant_id                     | UUID         | Multi-tenant identifier |
| doctor_code                   | VARCHAR(50)  | Unique doctor code      |
| employee_id                   | UUID         | Employee reference      |
| first_name                    | VARCHAR(100) | First name              |
| middle_name                   | VARCHAR(100) | Middle name             |
| last_name                     | VARCHAR(100) | Last name               |
| gender_id                     | BIGINT       | Gender reference        |
| email                         | VARCHAR(255) | Email address           |
| phone                         | VARCHAR(20)  | Contact number          |
| primary_specialization_id     | UUID         | Main specialization     |
| status_id                     | BIGINT       | Doctor status           |
| joining_date                  | DATE         | Joining date            |
| consultation_experience_years | INT          | Experience in years     |
| created_at                    | TIMESTAMP    | Record creation time    |
| updated_at                    | TIMESTAMP    | Last update time        |
| created_by                    | UUID         | Created user            |
| updated_by                    | UUID         | Updated user            |
| deleted_at                    | TIMESTAMP    | Soft delete timestamp   |
