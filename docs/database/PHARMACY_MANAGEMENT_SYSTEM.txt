# Enterprise Hospital Management System (HMS)

# Pharmacy Management System Documentation

---

# 1. Module Overview

The Pharmacy Management System is the medication, prescription, and drug inventory backbone of the hospital management system.

This module manages:

* Drug master management
* Prescription workflows
* E-prescriptions
* Medication dispensing
* Drug interaction checking
* Inventory management
* Batch tracking
* Expiry management
* Procurement and suppliers
* Controlled substances
* Cold chain monitoring
* Pharmacy billing
* Insurance claims
* Audit and compliance

This module integrates with:

* Patient Management
* Doctor Management
* EMR/EHR
* Billing
* Insurance
* Inventory systems
* Barcode systems
* Smart cabinets
* Supplier systems

---

# 2. Enterprise Architecture Principles

This schema follows enterprise pharmacy standards:

* UUID-based primary keys
* Normalized pharmacy architecture
* Multi-store support
* Batch-level inventory tracking
* Expiry and cold-chain monitoring
* Controlled substance compliance
* E-prescription workflows
* Audit-ready architecture
* Enterprise scalability
* Multi-hospital support

# TABLE: drugs

## Purpose

This is the core drug master table.

## Why This Table Exists

Stores all medicine information.

* One drug can have multiple interactions.
* One drug can exist in inventory.
* One drug can appear in prescriptions.

| drug_id                 | UUID         | Primary key              |

| generic_name            | VARCHAR(255) | Generic medicine name    |

| drug_category_id        | UUID         | FK to drug_categories    |
| drug_subcategory_id     | UUID         | FK to drug_subcategories |
| drug_form_id            | UUID         | FK to drug_forms         |
| drug_unit_id            | UUID         | FK to drug_units         |

| description             | TEXT         | Additional details       |
| created_at              | TIMESTAMP    | Creation timestamp       |
