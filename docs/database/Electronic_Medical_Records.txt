# Enterprise Hospital Management System (HMS)

# EMR / EHR (Electronic Medical Records) Module Documentation

---

# 1. Module Overview

The EMR / EHR module is the clinical core of the Hospital Management System.

This module manages:

* Patient encounters
* Clinical notes
* Diagnoses
* Symptoms
* Vital signs
* Care plans
* Treatment plans
* Medication records
* Nursing observations
* Procedures
* Immunizations
* Referrals
* Discharge summaries
* Clinical alerts
* Clinical documents
* Audit tracking

This module integrates with:

* Patient Management
* Doctor Management
* Appointment Management
* Admission Management
* Pharmacy
* Laboratory
* Radiology
* Billing
* Nursing
* Telemedicine

---

# 2. Enterprise Architecture Principles

This schema follows enterprise-grade healthcare standards:

* UUID-based primary keys
* Normalized relational architecture
* Audit logging support
* Change history tracking
* HIPAA/GDPR-friendly design
* Multi-tenant architecture
* Scalable encounter management
* Clinical workflow support
* High-volume medical record support
* Integration-ready architecture

# 4. Core Clinical Tables

---

# TABLE: patient_encounters

## Purpose

This is the root table of the EMR/EHR module.

## Why This Table Exists

Stores all clinical patient encounters.

## Relationships

* One patient can have many encounters.
* One encounter can have many diagnoses.
* One encounter can have many notes.
* One encounter can have many procedures.
* One encounter can have many medications.

---

| Column Name       | Data Type    | Description             |
| ----------------- | ------------ | ----------------------- |
| encounter_id      | UUID         | Primary key             |
| tenant_id         | UUID         | Multi-tenant identifier |
| encounter_number  | VARCHAR(100) | Unique encounter number |
| patient_id        | UUID         | FK to patients          |
| appointment_id    | UUID         | FK to appointments      |
| doctor_id         | UUID         | FK to doctors           |
| department_id     | UUID         | FK to departments       |
| encounter_type_id | UUID         | FK to encounter_types   |
| encounter_date    | TIMESTAMP    | Encounter timestamp     |
| chief_complaint   | TEXT         | Main patient complaint  |
| clinical_summary  | TEXT         | Clinical overview       |
| encounter_status  | VARCHAR(100) | Encounter state         |
| created_at        | TIMESTAMP    | Creation timestamp      |
| updated_at        | TIMESTAMP    | Update timestamp        |
| created_by        | UUID         | Created user            |
| updated_by        | UUID         | Updated user            |
| deleted_at        | TIMESTAMP    | Soft delete timestamp   |

---
