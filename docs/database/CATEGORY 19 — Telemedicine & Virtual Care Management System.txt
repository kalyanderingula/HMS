# CATEGORY 19 — TELEMEDICINE & VIRTUAL CARE MANAGEMENT SYSTEM

## Module Purpose

The Telemedicine & Virtual Care Management System enables hospitals to provide remote healthcare services through video consultations, remote patient monitoring, electronic prescriptions, patient portals, and virtual care workflows.

This module integrates with:

* Patient Management
* Doctor Management
* Appointment Management
* EMR/EHR
* Billing System
* Pharmacy System
* Insurance Claims
* Mobile Applications

---

# 1. telemedicine_specialties

## Purpose

Stores telemedicine medical specialties offered by the hospital.

## Example

* Tele Cardiology
* Tele Dermatology
* Tele Psychiatry

| Column         | Data Type    | Description           |
| -------------- | ------------ | --------------------- |
| specialty_id   | UUID         | Primary Key           |
| specialty_code | VARCHAR(50)  | Unique specialty code |
| specialty_name | VARCHAR(255) | Specialty name        |
| description    | TEXT         | Description           |
| created_at     | TIMESTAMP    | Record creation date  |

---

# 3. telemedicine_providers

## Purpose

Stores doctors approved for telemedicine services.

## Relationships

* Many providers belong to one specialty.
* One provider can have many appointments.

| Column              | Data Type   | Description                    |
| ------------------- | ----------- | ------------------------------ |
| provider_id         | UUID        | Primary Key                    |
| doctor_id           | UUID        | Doctor reference               |
| specialty_id        | UUID        | FK to telemedicine_specialties |
| provider_status     | VARCHAR(50) | Active/Inactive                |
| years_of_experience | INT         | Experience                     |
| created_at          | TIMESTAMP   | Record creation                |
