# CATEGORY 22 — SECURITY, IAM & COMPLIANCE MANAGEMENT SYSTEM

## Module Overview

The Security, Identity & Access Management (IAM), and Compliance Management System is the security foundation of the entire Hospital Management System (HMS).

This module controls:

* User Authentication
* User Authorization
* Role-Based Access Control (RBAC)
* Multi-Factor Authentication (MFA)
* API Security
* Audit Logging
* Compliance Management
* Risk Management
* Data Protection
* Encryption
* Security Monitoring

This module integrates with every HMS module.

---

# 1. USER MANAGEMENT

## TABLE: users

### Purpose

Stores all system users.

### Examples

* Doctors
* Nurses
* Administrators
* Receptionists
* Patients
* Vendors

| Column        | Data Type    | Description        |
| ------------- | ------------ | ------------------ |
| user_id       | UUID         | Primary Key        |
| username      | VARCHAR(255) | Login username     |
| email         | VARCHAR(255) | Email              |
| password_hash | TEXT         | Encrypted password |
| status        | VARCHAR(50)  | Active/Inactive    |
| created_at    | TIMESTAMP    | Creation date      |

---

## TABLE: user_profiles

### Purpose

Stores detailed user profile information.

### Relationship

One user → One profile

| Column     | Data Type    | Description    |
| ---------- | ------------ | -------------- |
| profile_id | UUID         | Primary Key    |
| user_id    | UUID         | User reference |
| first_name | VARCHAR(255) | First name     |
| last_name  | VARCHAR(255) | Last name      |
| phone      | VARCHAR(50)  | Phone          |

---

## TABLE: user_sessions

### Purpose

Tracks active user sessions.

### Examples

* Web Login
* Mobile App Login

| Column      | Data Type | Description    |
| ----------- | --------- | -------------- |
| session_id  | UUID      | Primary Key    |
| user_id     | UUID      | User reference |
| login_time  | TIMESTAMP | Login time     |
| logout_time | TIMESTAMP | Logout time    |

---

## TABLE: login_history

### Purpose

Stores login audit history.

### Security Benefits

* Detect suspicious activity
* Audit access history

| Column           | Data Type    | Description     |
| ---------------- | ------------ | --------------- |
| login_history_id | UUID         | Primary Key     |
| user_id          | UUID         | User reference  |
| ip_address       | VARCHAR(255) | Source IP       |
| login_time       | TIMESTAMP    | Login timestamp |

# 3. PASSWORD SECURITY

## TABLE: password_policies

### Purpose

Defines password standards.

### Examples

* Minimum length
* Expiration days
* Complexity requirements

| Column      | Data Type | Description       |
| ----------- | --------- | ----------------- |
| policy_id   | UUID      | Primary Key       |
| min_length  | INT       | Password length   |
| expiry_days | INT       | Expiration period |

---

## TABLE: password_history

### Purpose

Prevents password reuse.

| Column        | Data Type | Description       |
| ------------- | --------- | ----------------- |
| history_id    | UUID      | Primary Key       |
| user_id       | UUID      | User              |
| password_hash | TEXT      | Previous password |
| changed_at    | TIMESTAMP | Change date       |

---

## TABLE: account_lockouts

### Purpose

Locks accounts after failed login attempts.

### Security Benefit

Prevents brute-force attacks.

| Column     | Data Type | Description |
| ---------- | --------- | ----------- |
| lockout_id | UUID      | Primary Key |
| user_id    | UUID      | User        |
| locked_at  | TIMESTAMP | Lock time   |
