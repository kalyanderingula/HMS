# Enterprise Hospital Management System (HMS)

# Inventory & Procurement Management System Documentation

---

# 1. Module Overview

The Inventory & Procurement Management System is the hospital’s centralized supply chain, warehouse management, procurement, vendor management, and inventory control backbone.

This module manages:

* Inventory categories and items
* Warehouse and storage management
* Inventory stock tracking
* Batch and expiry management
* Procurement workflows
* Purchase orders and goods receipts
* Vendor management
* Asset management
* Biomedical equipment tracking
* Cold chain monitoring
* Controlled substance inventory
* Hazardous material compliance
* Reorder automation
* Inventory analytics
* Audit and compliance tracking

This module integrates with:

* Pharmacy Management
* Laboratory Information System
* Surgery & OT
* ICU & Critical Care
* CSSD
* Billing & Financial Systems
* ERP Systems
* Biomedical Engineering
* Barcode/RFID Systems
* IoT Sensors

---

# 2. Enterprise Architecture Principles

This schema follows enterprise inventory and supply chain standards:

* UUID-based primary keys
* Multi-warehouse architecture
* Batch and expiry tracking
* Real-time inventory movement tracking
* Automated procurement workflows
* Barcode and RFID readiness
* Audit-ready design
* High-volume inventory scalability
* Multi-hospital inventory support
* Enterprise-grade compliance architecture

# TABLE: inventory_items

## Purpose

Stores inventory item master data.

## Why This Table Exists

Acts as the central inventory catalog.

## Relationships

* One item can have stock records.
* One item can have procurement records.
* One item can have expiry tracking.
* One item can have barcode mappings.

| Column Name                 | Data Type    | Description                      |
| --------------------------- | ------------ | -------------------------------- |
| inventory_item_id           | UUID         | Primary key                      |

| item_type                   | VARCHAR(100) | Consumable/Asset/etc             |
| unit_of_measure_id          | UUID         | FK to inventory_units_of_measure |

| is_batch_tracking_required  | BOOLEAN      | Batch tracking flag              |

## Why This Table Exists

Supports:

* Inventory traceability
* Regulatory audits
* Supply chain investigations
* Compliance reviews

| Column Name        | Data Type    | Description       |
| ------------------ | ------------ | ----------------- |
| audit_log_id       | UUID         | Primary key       |
| entity_name        | VARCHAR(255) | Table/entity name |
| entity_id          | UUID         | Record reference  |
| action_type        | VARCHAR(100) | Action type       |
| action_description | TEXT         | Action details    |
| performed_by       | UUID         | User reference    |
| performed_at       | TIMESTAMP    | Action timestamp  |

# TABLE: inventory_change_history

## Purpose

Tracks field-level inventory changes.

| Column Name       | Data Type    | Description        |
| ----------------- | ------------ | ------------------ |
| change_history_id | UUID         | Primary key        |
| table_name        | VARCHAR(255) | Modified table     |
| record_id         | UUID         | Modified record    |
| changed_field     | VARCHAR(255) | Changed field      |
| old_value         | TEXT         | Previous value     |
| new_value         | TEXT         | Updated value      |
| changed_by        | UUID         | User making change |
| changed_at        | TIMESTAMP    | Change timestamp   |
