# Enterprise Hospital Management System (HMS)

# Radiology Information System (RIS) & PACS Documentation

---

# 1. Module Overview

The Radiology Information System (RIS) & PACS module is the imaging and diagnostic visualization backbone of the hospital management system.

This module manages:

* Radiology orders
* Imaging workflows
* DICOM image storage
* PACS archival
* Imaging reports
* Radiologist workflows
* Radiation dose tracking
* Contrast administration
* AI image analysis
* Radiology billing
* Modality integration
* Audit and compliance

This module integrates with:

* Patient Management
* EMR/EHR
* Billing
* Laboratory
* Insurance
* PACS systems
* DICOM systems
* HL7/FHIR integrations
* AI diagnostic platforms

---

# 2. Enterprise Architecture Principles

This schema follows enterprise radiology standards:

* UUID-based primary keys
* DICOM-compatible architecture
* PACS-ready imaging structure
* Multi-modality support
* Multi-site imaging support
* AI-assisted diagnostics
* Audit-ready architecture
* Radiation safety tracking
* Cloud imaging scalability
* Enterprise interoperability

# 6. Radiology Orders

# TABLE: radiology_orders

## Purpose

This is the root table of RIS operations.

## Why This Table Exists

Stores all imaging orders.

## Relationships

* One order can contain multiple imaging tests.
* One order can generate multiple studies.
* One order can produce radiology reports.

| Column Name               | Data Type    | Description                    |
| ------------------------- | ------------ | ------------------------------ |
| radiology_order_id        | UUID         | Primary key                    |
| order_number              | VARCHAR(100) | Unique order number            |
| patient_id                | UUID         | FK to patients                 |
| encounter_id              | UUID         | FK to patient_encounters       |
| doctor_id                 | UUID         | FK to doctors                  |
| radiology_order_status_id | UUID         | FK to radiology_order_statuses |
| priority_id               | UUID         | FK to radiology_priorities     |
| clinical_indication       | TEXT         | Clinical reason                |
| ordered_at                | TIMESTAMP    | Order timestamp                |
| created_by                | UUID         | Ordering user                  |
| created_at                | TIMESTAMP    | Creation timestamp             |
